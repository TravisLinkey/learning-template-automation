Welcome to Learn Networking for Quant Systems! This tutorial series is designed to help you understand networking fundamentals, which are essential for quantitative finance since markets are networked systems.

## What is Networking?

**Computer networking** is the practice of interfacing two or more computing devices with each other for the purpose of sharing data. Learn more: [Computer Network](https://en.wikipedia.org/wiki/Computer_network). In quantitative finance, networking is crucial for:

- **Market Data**: Receiving real-time market data feeds
- **Order Execution**: Sending orders to exchanges
- **Low Latency**: Minimizing network latency for HFT
- **System Architecture**: Understanding distributed systems
- **Infrastructure**: Understanding market infrastructure and connectivity

## Why Learn Networking for Quant Development?

As a quantitative developer, you'll work with:

1. **Protocols**: Understanding TCP vs UDP and when to use each
2. **Latency**: Understanding what causes network latency
3. **Throughput**: Balancing latency and throughput
4. **Serialization**: Understanding serialization overhead
5. **Low-Latency Networking**: Techniques for minimizing latency in HFT
