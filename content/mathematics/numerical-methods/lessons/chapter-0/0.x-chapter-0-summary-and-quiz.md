**Question #2**

Why is understanding floating point arithmetic important for numerical computing?

<details>
<summary>Show Solution</summary>

Understanding floating point arithmetic is important because:

1. **Limited Precision**: Floating point numbers have finite precision, leading to rounding errors
2. **Error Accumulation**: Small errors can accumulate in iterative algorithms
3. **Numerical Stability**: Some algorithms are more stable than others
4. **Avoiding Pitfalls**: Understanding helps avoid common mistakes like comparing floats for equality
5. **Performance**: Understanding can help optimize numerical code

Without understanding floating point, you may write code that produces incorrect results or fails unexpectedly.

</details>

**Question #4**

What is the difference between symbolic manipulation and numerical approximation?

<details>
<summary>Show Solution</summary>

- **Symbolic Manipulation**: Solving problems exactly using mathematical symbols and formulas. Example: Solving $x^2 - 5x + 6 = 0$ to get exact solutions $x = 2$ or $x = 3$.

- **Numerical Approximation**: Solving problems approximately using numerical algorithms. Example: Finding roots of $f(x) = 0$ using iterative methods like Newton's method, which gives approximate solutions.

Numerical methods use approximation because many problems can't be solved symbolically, or symbolic solutions are too complex. We trade exactness for computability and practicality.

</details>
