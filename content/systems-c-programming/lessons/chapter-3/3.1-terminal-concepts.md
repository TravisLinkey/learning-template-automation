# 3.1 — Terminal Concepts (TTY vs PTY)

This chapter covers terminal concepts needed to build a terminal emulator. Understanding TTY vs PTY is essential for creating interactive applications.

## What is a Terminal?

A terminal is an interface for text input/output. Historically, it was a physical device (teletype), now it's emulated in software.

### Key Components

| Component | Description |
|-----------|-------------|
| **TTY** | Physical or emulated terminal device |
| **PTY** | Pseudo-terminal (pair of virtual devices) |
| **Shell** | Command interpreter (bash, zsh, sh) |
| **Termcap/Terminfo** | Terminal capability database |

## TTY (Teletype)

The original Unix terminals were teletypes — keyboards and printers. The term "TTY" persists as shorthand for terminal devices.

### Terminal Device Files

```bash
ls -la /dev/tty
ls -la /dev/pts/
```

- `/dev/tty` — Controlling terminal of the process
- `/dev/pts/*` — Pseudo-terminal slave devices

## PTY (Pseudo-Terminal)

A PTY is a pair of virtual devices:
- **Master** — The controlling end (your application)
- **Slave** — The "fake" terminal (shell sees this as its TTY)

```
┌─────────────────────────────────────────────────────┐
│                   Your Application                   │
│                      (PTY Master)                    │
└────────────────────────┬────────────────────────────┘
                         │ read()/write()
┌────────────────────────▼────────────────────────────┐
│                   Shell / Program                     │
│                      (PTY Slave)                      │
└─────────────────────────────────────────────────────┘
```

## Creating a PTY

```c
#include <stdlib.h>
#include <unistd.h>

// Open pseudo-terminal
int posix_openpt(int oflag);

// Grant access to slave
int grantpt(int fd);

// Unlock slave
int unlockpt(int fd);

// Get slave name
char *ptsname(int fd);

// Or use openpty() convenience function
int openpty(int *amaster, int *aslave, char *name, struct termios *termp, struct winsize *winp);
```

### Example: Basic PTY Setup

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pty.h>
#include <utmp.h>

int master, slave;
char *slave_name;

if (openpty(&master, &slave, NULL, NULL, NULL) < 0) {
    perror("openpty");
    return 1;
}

slave_name = ptsname(master);
printf("Master: %d, Slave: %d (%s)\n", master, slave, slave_name);
```

## PTY for Terminal Emulation

A terminal emulator needs to:
1. Create a PTY pair
2. Spawn a shell in the slave
3. Read from master and display to user
4. Send user input to master

```c
// Simplified terminal emulator flow

// 1. Create PTY
openpty(&master, &slave, ...);

// 2. Fork
pid = fork();

if (pid == 0) {
    // Child: set up slave as controlling terminal
    close(master);
    setsid();  // Create new session
    ioctl(slave, TIOCSCTTY, 0);  // Set controlling terminal
    
    // Redirect stdin/stdout/stderr to slave
    dup2(slave, 0);
    dup2(slave, 1);
    dup2(slave, 2);
    close(slave);
    
    // Execute shell
    execl("/bin/bash", "bash", NULL);
}

// Parent: communicate with master
// Read from master, display to user
// Read from user, write to master
```

## Session and Process Group

### Session Leader

```c
#include <unistd.h>

pid_t setsid(void);  // Create new session, become session leader
pid_t getsid(pid_t pid);  // Get session ID
```

### Process Group

```c
#include <unistd.h>

pid_t getpgrp(void);  // Get process group
int setpgid(pid_t pid, pid_t pgid);  // Set process group
```

## TIOCSCTTY — Set Controlling Terminal

To make a PTY slave the controlling terminal:

```c
#include <sys/ioctl.h>

ioctl(slave_fd, TIOCSCTTY, 0);
```

This only works if the process is the session leader.

## Terminal Attributes (termios)

```c
#include <termios.h>

struct termios {
    tcflag_t c_iflag;  // Input modes
    tcflag_t c_oflag;  // Output modes
    tcflag_t c_cflag;  // Control modes
    tcflag_t c_lflag;  // Local modes
    cc_t c_cc[NCCS];   // Control characters
};
```

### Getting/Setting Attributes

```c
struct termios orig_termios;

tcgetattr(STDIN_FILENO, &orig_termios);
tcsetattr(STDIN_FILENO, TCSANOW, &new_termios);
```

### Common Settings

```c
// Raw mode (disable echo, canonical mode, etc.)
struct termios raw;
tcgetattr(fd, &raw);
cfmakeraw(&raw);
tcsetattr(fd, TCSAFLUSH, &raw);
```

## Window Size (winsize)

```c
#include <sys/ioctl.h>

struct winsize {
    unsigned short ws_row;    // rows
    unsigned short ws_col;     // columns
    unsigned short ws_xpixel;  // pixels
    unsigned short ws_ypixel;
};

// Get
ioctl(fd, TIOCGWINSZ, &winsize);

// Set
ioctl(fd, TIOCSWINSZ, &winsize);
```

## Signal Handling in Terminals

Common terminal signals:

| Signal | Key | Description |
|--------|-----|-------------|
| `SIGINT` | Ctrl+C | Interrupt |
| `SIGQUIT` | Ctrl+\\ | Quit |
| `SIGTSTP` | Ctrl+Z | Suspend |
| `SIGTTIN` | | Background read |
| `SIGTTOU` | | Background write |
