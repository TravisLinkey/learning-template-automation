# 2.2 — Standard I/O Streams

C provides three standard streams: stdin, stdout, and stderr. Understanding them is key to building interactive programs and command-line tools.

## Standard Streams

| Stream | Description | Default | File Descriptor |
|--------|-------------|---------|------------------|
| `stdin` | Standard input | Keyboard | 0 |
| `stdout` | Standard output | Screen | 1 |
| `stderr` | Standard error | Screen | 2 |

```c
#include <stdio.h>

// stdin, stdout, stderr are pre-defined FILE* pointers
fprintf(stdout, "Output to stdout\n");
fprintf(stderr, "Error to stderr\n");
```

## Using stdin

### Simple Input

```c
int num;
printf("Enter a number: ");
scanf("%d", &num);

// Or use fgets + sscanf for safer input
char buffer[100];
fgets(buffer, sizeof(buffer), stdin);
sscanf(buffer, "%d", &num);
```

### Reading from File as stdin

```c
FILE *fp = fopen("input.txt", "r");
if (fp != NULL) {
    // Temporarily redirect stdin
    FILE *old_stdin = stdin;
    stdin = fp;
    
    // Now scanf reads from file
    int num;
    scanf("%d", &num);
    
    // Restore stdin
    stdin = old_stdin;
    fclose(fp);
}
```

## stdout vs stderr

### Why Separate Them?

- **stdout** — Normal output (can be redirected/piped)
- **stderr** — Error messages (often goes to screen even if stdout is redirected)

```c
fprintf(stdout, "Processing...\n");
fprintf(stderr, "Error: Something went wrong\n");
```

```bash
# Redirect stdout to file, stderr still goes to screen
./program > output.txt

# Redirect both
./program > output.txt 2> errors.txt

# Redirect stderr to stdout
./program > output.txt 2>&1
```

## Redirection

### Command Line Redirection

```bash
# Read from file instead of keyboard
./program < input.txt

# Write to file instead of screen
./program > output.txt

# Append to file
./program >> output.txt

# Pipe: stdout of one program to stdin of another
./producer | ./consumer
```

### Example: Simple Filter

```c
// upper.c - convert input to uppercase
#include <stdio.h>
#include <ctype.h>

int main(void) {
    int c;
    while ((c = getchar()) != EOF) {
        putchar(toupper(c));
    }
    return 0;
}
```

```bash
# Usage
./upper < input.txt          # Read from file
echo "hello" | ./upper       # Pipe from echo
./upper > output.txt         # Write to file
```

## Buffering

Streams can have different buffering modes:

| Mode | Description |
|------|-------------|
| `_IONBF` | No buffering |
| `_IOLBF` | Line buffered |
| `_IOFBF` | Fully buffered |

```c
// Set buffering
setvbuf(stdout, NULL, _IONBF, 0);  // No buffering

// Or when opening file
FILE *fp = fopen("file.txt", "w");
setvbuf(fp, NULL, _IOFBF, 4096);   // Fully buffered, 4KB buffer
```

### When Buffering Matters

```c
// Without flushing, this might not print immediately
printf("Processing...");
sleep(5);  // User won't see message until after sleep or flush

// Force flush
printf("Processing...\n");  // '\n' flushes for stdout with line buffering
// Or
printf("Processing...");
fflush(stdout);  // Explicit flush
```

## Putting It Together: CLI Tool

```c
#include <stdio.h>
#include <stdlib.h>

int main(int argc, char *argv[]) {
    if (argc != 2) {
        fprintf(stderr, "Usage: %s <filepath>\n", argv[0]);
        return 1;
    }
    
    FILE *fp = fopen(argv[1], "r");
    if (fp == NULL) {
        fprintf(stderr, "Error: Cannot open %s\n", argv[1]);
        return 1;
    }
    
    int line_count = 0;
    char buffer[1024];
    
    while (fgets(buffer, sizeof(buffer), fp) != NULL) {
        line_count++;
    }
    
    fprintf(stdout, "%s: %d lines\n", argv[1], line_count);
    
    fclose(fp);
    return 0;
}
```
