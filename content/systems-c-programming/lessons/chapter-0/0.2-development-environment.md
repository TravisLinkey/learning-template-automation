# 0.2 — Development Environment

Before writing C code, you need to set up your development environment. This lesson covers installing the C compiler, build tools, and configuring your editor.

## The C Toolchain

```
Editor → Preprocessor → Compiler → Assembler → Linker → Executable
```

| Tool | Purpose |
|------|---------|
| **gcc/clang** | The C compiler |
| **make** | Build automation |
| **gdb/lldb** | Debugger |
| **valgrind** | Memory debugging |

## Installing the C Compiler

### macOS
```bash
# Xcode Command Line Tools (includes clang)
xcode-select --install

# Or install GCC via Homebrew
brew install gcc
```

### Linux (Ubuntu/Debian)
```bash
sudo apt update
sudo apt install build-essential
```

### Linux (Fedora/RHEL)
```bash
sudo dnf install gcc make
```

### Windows
```bash
# Via Chocolatey
choco install mingw

# Or use WSL (Windows Subsystem for Linux)
# Install Ubuntu from Microsoft Store, then follow Linux instructions
```

## Verifying Your Installation

```bash
# Check compiler version
gcc --version

# Check make version
make --version

# Example output:
# gcc (Ubuntu 11.4.0-1ubuntu1~22.04) 11.4.0
# GNU Make 4.3
```

## Your First C Program

Create a file called `hello.c`:

```c
#include <stdio.h>

int main(void) {
    printf("Hello, World!\n");
    return 0;
}
```

## Compiling and Running

```bash
# Compile (creates executable named 'hello')
gcc hello.c -o hello

# Run
./hello

# Output:
# Hello, World!
```

## Compilation Flags

| Flag | Description |
|------|-------------|
| `-Wall` | Enable all warnings |
| `-Wextra` | Enable extra warnings |
| `-g` | Include debug symbols |
| `-O2` | Optimize for speed |
| `-O0` | No optimization (for debugging) |
| `-o file` | Name output file |

```bash
# Recommended for development
gcc -Wall -Wextra -g hello.c -o hello

# Recommended for release
gcc -O2 -Wall hello.c -o hello
```

## Using Make

For larger projects, use Make to automate the build process. Create a `Makefile`:

```makefile
# Makefile for hello program

CC = gcc
CFLAGS = -Wall -Wextra -g
TARGET = hello
SRC = hello.c

$(TARGET): $(SRC)
	$(CC) $(CFLAGS) $(SRC) -o $(TARGET)

clean:
	rm -f $(TARGET)

.PHONY: clean
```

Build and clean:

```bash
make          # Build the project
make clean   # Remove built files
```

## Editor Configuration

### VS Code
Install the "C/C++" extension by Microsoft. It provides:
- IntelliSense (code completion)
- Debugging support (GDB/LLDB)
- Code formatting

### Vim/Neovim
Use these plugins:
- `vim-polyglot` — Syntax highlighting
- `coc-clangd` — LSP for C
- Or use built-in C syntax highlighting

### Emacs
Use `cc-mode` for C editing with proper indentation and formatting.

## Debugging with GDB

```bash
# Compile with debug symbols
gcc -g hello.c -o hello

# Start debugger
gdb ./hello

# Common commands:
# run        — Run the program
# break main — Set breakpoint at main
# next       — Execute next line
# print var  — Print variable value
# quit       — Exit debugger
```

## Memory Debugging with Valgrind

```bash
# Install valgrind
brew install valgrind  # macOS
sudo apt install valgrind  # Linux

# Run memory check
valgrind --leak-check=full ./hello
```

This catches memory leaks, invalid memory access, and other memory-related bugs.
