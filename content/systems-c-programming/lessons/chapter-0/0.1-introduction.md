# 0.1 — Introduction to C

C is a low-level programming language that gives you direct access to system resources like memory, file descriptors, and processes. It's the language behind operating systems, embedded systems, and high-performance applications.

## Why Learn C?

| Reason | Description |
|--------|-------------|
| **Understanding Systems** | C teaches you how computers actually work under the hood |
| **Performance** | No runtime overhead — your code runs directly on the hardware |
| **Portability** | C compiles to virtually any platform with a C compiler |
| **Foundation for Other Languages** | C++, Objective-C, and many others are C-based |
| **Systems Programming** | OS kernels, drivers, embedded systems, and terminals |

## What C Gives You Access To

- **Memory** — Direct pointer access to RAM
- **Files** — File descriptors and low-level I/O
- **Processes** — Create and manage child processes
- **Hardware** — Memory-mapped I/O, CPU registers
- **Networking** — Sockets, protocols, low-level networking

## C vs Other Languages

```
Python:     interpreter → bytecode → VM → execution
Java:       compiler → bytecode → JVM → execution
C:          compiler → machine code → direct execution
```

C compiles directly to machine code. There's no virtual machine or interpreter between your code and the CPU.

## What You'll Build

This subject teaches C fundamentals that culminate in building a **terminal emulator** using PTY (pseudo-terminals). You'll learn:

1. **C syntax and data types** — Variables, functions, control flow
2. **Pointers and memory** — The heart of C programming
3. **File I/O and processes** — System-level operations
4. **PTY concepts** — Building an interactive terminal

- Familiarity with command line (terminal/shell)

## History Brief

C was developed in 1972 at Bell Labs by Dennis Ritchie. It was designed to write the Unix operating system. The language was formalized in 1989 as ANSI C (C89) and later standardized as ISO C (C99, C11, C17, C23).

Key milestones:
- **1972** — C created at Bell Labs
- **1978** — K&R C (The C Programming Language book)
- **1989** — ANSI C (C89)
- **1999** — C99 (new features like variable-length arrays)
- **2011** — C11 (multithreading support)
- **2018** — C17 (bug fixes, no new features)
- **2023** — C23 (latest standard)

Most systems today use C11 or later.

## The C Ecosystem

```
Source (.c) → Compiler (gcc/clang) → Object (.o) → Linker → Executable
```

The build process:
1. **Preprocessor** — Handles #include, #define macros
2. **Compiler** — Translates C to assembly/object code
3. **Linker** — Combines object files and libraries into executable

## Self-Investigation Tools

Throughout these lessons, we'll reference many C library functions and POSIX system calls. Rather than including exhaustive explanations for each function, we encourage you to look up details yourself using these tools:

### man — Function Reference

The primary tool for investigating C functions. POSIX system calls and standard library functions have comprehensive man pages:

```bash
# Look up system calls directly (section 2)
man fork
man execve
man waitpid

# Look up library functions (section 3)
man fopen
man fprintf
man signal
```

| Section | Content |
|--------|---------|
| `man 1` | User commands (shell utilities) |
| `man 2` | System calls (kernel functions like `fork`, `read`, `write`) |
| `man 3` | C library functions (`fopen`, `printf`, `malloc`) |
| `man 7` | Miscellaneous (protocols, file formats) |

### man -k — Search by Keyword

Search for functions related to a concept:

```bash
# Find all process-related functions
man -k "process"

# Find functions matching a pattern
man -k "create process"
man -k "terminal control"
```

### Quick Reference Examples

```bash
# Get just the function signature from a man page (use less for navigation)
man 2 fork | head -30

# Search within a man page
/SEE ALSO
```

### What to Look For

When reading a function's man page, focus on:

1. **SYNOPSIS** — The function signature and required headers
2. **DESCRIPTION** — What the function does
3. **RETURN VALUE** — What the function returns on success/error
4. **ERRORS** — Error conditions you should handle
5. **SEE ALSO** — Related functions

This skill is essential for systems programming — you'd regularly reference man pages in practice.
