# 1.1 — Pointers and Addresses

Pointers are what make C powerful. They let you work directly with memory addresses, enabling efficient data structures, function callbacks, and systems programming.

## What is a Pointer?

A pointer is a variable that stores a **memory address** instead of a value. Every variable in C is stored at some location in memory, and pointers let you access that location directly.

```c
int num = 42;      // An integer variable
int *ptr = &num;   // A pointer to that integer
```

- `&num` — "Address of num"
- `int *` — "Pointer to int"
- `ptr` — Stores the address of `num`

## Visualizing Memory

```
Memory Address    Variable    Value
─────────────────────────────────────────
0x7fff5a3c8a00    num         42
0x7fff5a3c8a04    ptr         0x7fff5a3c8a00  (points to num)
```

## Declaring Pointers

```c
// Pointer to int
int *int_ptr;

// Pointer to char
char *char_ptr;

// Pointer to double
double *double_ptr;

// Pointer to void (generic)
void *generic_ptr;
```

The `*` can be attached to either the type or the variable:

```c
int* ptr1;   // Style 1: pointer is int*
int *ptr2;   // Style 2: pointer is to int (more common)
int *ptr3, ptr4;  // ptr3 is pointer, ptr4 is int!
```

## Getting Addresses

Use `&` (address-of operator) to get a variable's address:

```c
int num = 42;
printf("Value: %d\n", num);        // 42
printf("Address: %p\n", &num);     // 0x7fff5a3c8a00 (example)
```

Use `%p` format specifier for addresses.

## Dereferencing Pointers

Use `*` (dereference operator) to access the value at the address:

```c
int num = 42;
int *ptr = &num;

printf("%d\n", *ptr);   // 42 (value at address)
*ptr = 100;             // Change value at address
printf("%d\n", num);    // 100 (num is now 100)
```

## Example Walkthrough

```c
#include <stdio.h>

int main(void) {
    int num = 42;
    int *ptr = &num;
    
    printf("num value:    %d\n", num);
    printf("num address:  %p\n", (void*)&num);
    printf("ptr value:    %p\n", (void*)ptr);
    printf("ptr points to: %d\n", *ptr);
    
    *ptr = 100;
    printf("After *ptr = 100:\n");
    printf("num value:    %d\n", num);
    
    return 0;
}
```

Output:
```
num value:    42
num address:  0x7fff5a3c8a00
ptr value:    0x7fff5a3c8a00
ptr points to: 42
After *ptr = 100:
num value:    100
```

## Pointer to Pointer

You can have a pointer to a pointer:

```c
int num = 42;
int *ptr1 = &num;      // pointer to int
int **ptr2 = &ptr1;    // pointer to pointer to int

printf("%d\n", **ptr2);  // 42
```

## Null Pointers

Always initialize pointers. Use `NULL` for "no address":

```c
int *ptr = NULL;  // Points to nothing

if (ptr != NULL) {
    printf("%d\n", *ptr);
}
```

## Pointers and Functions

### Passing by Pointer

Pass the address to allow a function to modify the original value:

```c
#include <stdio.h>

void double_value(int *ptr) {
    *ptr = *ptr * 2;
}

int main(void) {
    int num = 21;
    double_value(&num);  // Pass address
    printf("%d\n", num); // 42
    return 0;
}
```

Compare to pass-by-value (without pointers):

```c
void double_value_wrong(int val) {
    val = val * 2;  // Only modifies local copy
}
```

## Why Pointers Matter

| Use Case | Example |
|----------|---------|
| **Dynamic memory** | `malloc()` returns a pointer |
| **Arrays** | Array names decay to pointers |
| **Function arguments** | Modify multiple values |
| **Data structures** | Linked lists, trees use pointers |
| **Systems programming** | File descriptors, memory-mapped I/O |
| **Performance** | Avoid copying large data |

## Common Pointer Mistakes

### 1. Uninitialized pointer
```c
int *ptr;      // Garbage address!
*ptr = 42;     // CRASH - writing to random memory
```

**Fix:** Initialize or allocate:
```c
int *ptr = NULL;
// or
int *ptr = malloc(sizeof(int));
```

### 2. Forgetting to free
```c
int *ptr = malloc(sizeof(int));
// ... use ptr ...
// Forgot to free!
// Memory leak!
```

### 3. Dangling pointer (use-after-free)
```c
int *ptr = malloc(sizeof(int));
free(ptr);
printf("%d\n", *ptr);  // CRASH - freed memory
```
