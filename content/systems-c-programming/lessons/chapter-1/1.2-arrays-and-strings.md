# 1.2 — Arrays and Strings

Arrays and strings are fundamental data structures in C. Understanding their relationship to pointers is essential for effective C programming.

## Declaring Arrays

```c
// Array of 10 integers
int numbers[10];

// Array with initialization
int primes[] = {2, 3, 5, 7, 11};

// Sized array with initialization
int scores[5] = {90, 85, 88, 92, 87};
```

## Array Indexing

```c
int scores[5] = {90, 85, 88, 92, 87};

scores[0] = 95;      // Set first element
printf("%d", scores[0]);  // 95
printf("%d", scores[4]);  // 87
```

**Important:** Array indices start at 0, not 1.

## Array and Pointer Relationship

In most contexts, an array **decays** to a pointer to its first element:

```c
int arr[5] = {1, 2, 3, 4, 5};
int *ptr = arr;  // Same as: int *ptr = &arr[0];

printf("%d\n", *ptr);     // 1 (first element)
printf("%d\n", *(ptr+1)); // 2 (second element)
printf("%d\n", ptr[2]);   // 3 (third element)
```

```
arr:     [1] [2] [3] [4] [5]
          ↑   ↑   ↑   ↑   ↑
address: 100 104 108 112 116  (assuming 4-byte int)

ptr = arr → points to 100
ptr + 1 → points to 104
ptr + 2 → points to 108
```

## Pointer Arithmetic

When you add to a pointer, it moves by the size of the pointed-to type:

```c
int arr[5] = {10, 20, 30, 40, 50};
int *ptr = arr;

printf("%d\n", *ptr);     // 10
printf("%d\n", *(ptr+1)); // 20
printf("%d\n", *(ptr+2)); // 30
```

This is equivalent to:
```c
printf("%d\n", ptr[0]);   // 10
printf("%d\n", ptr[1]);   // 20
printf("%d\n", ptr[2]);   // 30
```

## Iterating Arrays

### Using Index
```c
int arr[] = {1, 2, 3, 4, 5};
for (int i = 0; i < 5; i++) {
    printf("%d ", arr[i]);
}
```

### Using Pointer
```c
int arr[] = {1, 2, 3, 4, 5};
int *ptr = arr;
int *end = arr + 5;

while (ptr < end) {
    printf("%d ", *ptr);
    ptr++;
}
```

## Multidimensional Arrays

```c
// 3x3 matrix
int matrix[3][3] = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

// Access elements
matrix[0][0] = 1;
matrix[2][2] = 9;
```

Memory layout (row-major):
```
[0][0] [0][1] [0][2] [1][0] [1][1] [1][2] [2][0] [2][1] [2][2]
   1      2      3      4      5      6      7      8      9
```

## Strings in C

Strings in C are arrays of characters ending with a **null terminator** (`\0`):

```c
// These are equivalent:
char name[] = "Hello";
char name[] = {'H', 'e', 'l', 'l', 'o', '\0'};
```

### String Functions

The `<string.h>` library provides useful functions:

```c
#include <string.h>

char s1[] = "Hello";
char s2[] = "World";

// Length
strlen(s1);  // 5

// Copy
char dest[20];
strcpy(dest, s1);  // dest = "Hello"

// Concatenate
strcat(s1, s2);  // s1 = "HelloWorld"

// Compare
strcmp(s1, s2);  // 0 if equal, <0 or >0 for lexicographic
```

### String Input

```c
char name[50];

// Read up to 49 characters (leaves room for \0)
printf("Enter name: ");
scanf("%49s", name);

// Safer alternative with fgets
fgets(name, sizeof(name), stdin);
// Note: fgets includes the newline, use strcspn to remove it
name[strcspn(name, "\n")] = '\0';
```

## Passing Arrays to Functions

Arrays decay to pointers when passed to functions:

```c
#include <stdio.h>

void print_array(int *arr, int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main(void) {
    int nums[] = {1, 2, 3, 4, 5};
    print_array(nums, 5);  // nums decays to &nums[0]
    return 0;
}
```

Alternative syntax (equivalent):
```c
void print_array(int arr[], int size) { ... }
void print_array(int *arr, int size) { ... }
```

## Common Array Mistakes

### 1. Out of bounds access
```c
int arr[5];
arr[5] = 100;  // CRASH - index 5 is outside array!
```

### 2. Forgetting null terminator
```c
char str[5] = "Hello";  // WRONG - no room for \0
char str[6] = "Hello";   // CORRECT
```

### 3. Not passing size
```c
// Don't do this - size information is lost!
void print_array(int arr[]) {  // arr is just a pointer!
    // Can't determine array size here
}
```
