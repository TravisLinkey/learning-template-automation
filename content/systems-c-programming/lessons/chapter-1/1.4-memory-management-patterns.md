# 1.4 — Memory Management Patterns

This lesson covers essential patterns and best practices for writing robust C code that handles memory correctly.

## Always Pair malloc with free

Every allocation must have a corresponding deallocation:

```c
// Pattern: allocate, use, free
struct Person *person = malloc(sizeof(struct Person));
if (person == NULL) return NULL;

// Use person
person->age = 30;

// Always free
free(person);
person = NULL;  // Prevent dangling pointer
```

## Initialize Pointers to NULL

Always initialize pointers, preferably to NULL:

```c
int *ptr = NULL;  // Clearly indicates "no memory"

// Later, allocate
ptr = malloc(sizeof(int));
```

This allows simple checks:

```c
if (ptr != NULL) {
    // Safe to use
}
```

## Check All Allocations

Every malloc/calloc/realloc can fail:

```c
int *ptr = malloc(sizeof(int));
if (ptr == NULL) {
    fprintf(stderr, "Failed to allocate memory\n");
    return ERROR_CODE;
}
```

## Free in Reverse Order

If allocating multiple resources, free in reverse order:

```c
// Allocate A, then B
int *a = malloc(sizeof(int));
char *b = malloc(100);

// Free in reverse: B first, then A
free(b);
free(a);
```

## Memory Ownership Patterns

### Caller Owns Return Value

```c
// Function allocates, caller frees
char *read_line(FILE *fp) {
    char *line = malloc(256);
    if (fgets(line, 256, fp) == NULL) {
        free(line);
        return NULL;
    }
    return line;  // Caller must free
}

// Usage:
char *line = read_line(fp);
if (line) {
    // use line
    free(line);  // Caller frees
}
```

### Caller Passes Buffer

```c
// Function uses caller's buffer (no allocation)
int read_line(FILE *fp, char *buffer, size_t size) {
    if (fgets(buffer, size, fp) == NULL) {
        return -1;
    }
    return 0;  // Caller's buffer, caller cleans up
}

// Usage:
char buffer[256];
if (read_line(fp, buffer, sizeof(buffer)) == 0) {
    // use buffer (no free needed)
}
```

### Transfer Ownership

```c
// Function allocates and returns ownership
typedef struct {
    int *data;
    size_t size;
} Buffer;

Buffer *buffer_create(size_t size) {
    Buffer *buf = malloc(sizeof(Buffer));
    if (buf == NULL) return NULL;
    buf->data = malloc(size * sizeof(int));
    if (buf->data == NULL) {
        free(buf);
        return NULL;
    }
    buf->size = size;
    return buf;
}

void buffer_destroy(Buffer *buf) {
    if (buf) {
        free(buf->data);
        free(buf);
    }
}

// Usage:
Buffer *buf = buffer_create(100);
// use buf
buffer_destroy(buf);  // Single cleanup function
```

## Use struct for Related Data

Group related data to ensure proper cleanup:

```c
typedef struct {
    int *array;
    size_t size;
    size_t capacity;
} DynamicArray;

// All in one allocation
DynamicArray *arr = malloc(sizeof(DynamicArray));
arr->array = NULL;
arr->size = 0;
arr->capacity = 0;

// Single free
free(arr->array);
free(arr);
```

## Const Correctness

Use `const` to prevent accidental modification:

```c
// Pointer to constant data
const int *ptr = &value;
// *ptr = 42;  // ERROR: cannot modify

// Constant pointer
int *const ptr = &value;
ptr = &other;  // ERROR: cannot change pointer

// Both
const int *const ptr = &value;
```

## Avoid sizeof on Pointers

Don't use sizeof on pointer variables:

```c
int *arr = malloc(100 * sizeof(int));

// WRONG: gives pointer size (8 bytes on 64-bit)
size_t size = sizeof(arr);

// CORRECT: gives element size
size_t size = sizeof(*arr);
// OR use the type
size_t size = sizeof(int);
```

## Cleanup on Error

Handle failures gracefully:

```c
int process_data(void) {
    int *a = NULL;
    int *b = NULL;
    int result = ERROR;
    
    a = malloc(sizeof(int));
    if (a == NULL) goto cleanup;
    
    b = malloc(sizeof(int));
    if (b == NULL) goto cleanup;
    
    // ... do work ...
    result = SUCCESS;
    
cleanup:
    free(a);
    free(b);
    return result;
}
```

## Valgrind Workflow

Always verify memory correctness:

```bash
# Compile with debug info
gcc -g -O0 program.c -o program

# Run valgrind
valgrind --leak-check=full --show-leak-kinds=all ./program

# Expected output with no leaks:
# All heap blocks were freed -- no leaks are possible
```

## Memory Debugging Checklist

- [ ] Every malloc has a corresponding free
- [ ] Every allocation is checked for NULL
- [ ] Pointers set to NULL after freeing
- [ ] No double-free errors
- [ ] No use-after-free
- [ ] No memory leaks (check with valgrind)
- [ ] Proper sizeof usage
