# 0.1 вҖ” Project 5: JSON Parser (Tree-sitter Style)

Build a JSON parser that tokenizes input, parses into an AST, and provides tree traversal.

## Objective

Create a JSON parser that:
1. **Tokenizes** JSON into a stream of tokens (lexer)
2. **Parses** tokens into an Abstract Syntax Tree (parser)
3. **Represents** JSON structure as a tree (AST)
4. **Traverses** the tree (tree-walker)

## JSON Token Types

| Token Type | Examples |
|------------|----------|
| `{` | `{` |
| `}` | `}` |
| `[` | `[` |
| `]` | `]` |
| `:` | `:` |
| `,` | `,` |
| **string** | `"hello"`, `"key"` |
| **number** | `42`, `3.14`, `-1e10` |
| **literal** | `true`, `false`, `null` |

## AST Node Types

```c
typedef enum {
    NODE_OBJECT,
    NODE_ARRAY,
    NODE_STRING,
    NODE_NUMBER,
    NODE_BOOLEAN,
    NODE_NULL,
} NodeType;

typedef struct Node {
    NodeType type;
    char *value;  // for strings/numbers
    int bool_value;  // for booleans
    struct Node **children;  // for objects/arrays
    size_t child_count;
    char **keys;  // for objects (key-value pairs)
} Node;
```

## Focus Areas

- **Lexer** вҖ” Character-by-character scanning to tokens
- **Recursive Descent Parser** вҖ” Handle JSON's recursive structure
- **AST Construction** вҖ” Build tree from parsed tokens
- **Tree Traversal** вҖ” Visit all nodes in the tree

## Why JSON?

- **Simple grammar** вҖ” Only 6 structural tokens + values
- **Recursive structure** вҖ” Objects and arrays can nest
- **Well-defined** вҖ” Clear spec, unambiguous parsing rules
- **Practical** вҖ” You can actually use the result
- **Foundational** вҖ” Same techniques apply to any language parser

## Technologies

- Any language (C, Python, etc.)
- No external parser libraries (build from scratch)

## Skills You'll Learn

1. **Lexing** вҖ” Convert text to tokens
2. **Token types** вҖ” Enums and token structs
3. **Recursive descent parsing** вҖ” Handle nested structures
4. **AST construction** вҖ” Build tree data structures
5. **Tree traversal** вҖ” Visit nodes depth-first

## Steps to Complete the Project

### 1. Define token types
Create an enum for token types and a Token struct.

**Hints:**
- Token types: LBRACE, RBRACE, LBRACKET, RBRACKET, COLON, COMMA, STRING, NUMBER, TRUE, FALSE, NULL, END
- Token struct: type, value (for strings/numbers), line, column

### 2. Implement the lexer
Convert input text to tokens.

**Hints:**
- Scan character by character
- Skip whitespace
- Handle strings (between `"` ...)
- Handle numbers (digit sequences, optional decimal, optional exponent)
- Handle literals (`true`, `false`, `null`)

### 3. Define AST nodes
Create node types for each JSON value type.

**Hints:**
- Use an enum for node type
- Each node stores its type and value
- Objects and arrays have children arrays
- Objects also store keys

### 4. Implement the parser
Parse tokens into AST.

**Hints:**
- Start with `parse_value()`
- `parse_value()` dispatches to `parse_object()`, `parse_array()`, or `parse_primitive()`
- Objects: expect `{`, parse key-value pairs, expect `}`
- Arrays: expect `[`, parse values, expect `]`
- Use recursion for nested structures

### 5. Implement tree traversal
Visit all nodes in the AST.

**Hints:**
- Depth-first traversal
- Callback function for each node visited
- Pass user data to callback
