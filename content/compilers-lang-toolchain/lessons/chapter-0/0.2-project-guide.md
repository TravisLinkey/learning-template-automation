This guide walks you through building the language toolchain by combining your terminal emulator and JSON parser.

## 1. Combine Projects

**Goal:** Integrate your terminal emulator and JSON parser into one build.

**Why it matters:** Having both components in one project lets them work together.

**Hints:**
- Copy JSON parser files to your terminal project
- Or add both to Makefile sources
- Start with working versions of each

**Try it:** Compile both together.

```makefile
%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

clean:
	rm -f $(TARGET) $(OBJECTS)

.PHONY: clean
```

## 3. Implement Command Handlers

**Goal:** Create functions for each command.

**Why it matters:** Commands are what make the tool useful.

**Hints:**
- `echo` — Print message from args
- `add/sub/mul/div` — Arithmetic
- `parse` — Test JSON parser on string
- `help` — List commands
- Extract args from AST node

**Try it:** Implement several commands.

<details>
<parameter name="Show Solution</summary>

```c
// Extract number from args by key
double get_number_arg(Node *args, const char *key) {
    for (size_t i = 0; i < args->child_count; i++) {
        if (strcmp(args->keys[i], key) == 0) {
            return args->children[i]->value.number_value;
        }
    }
    return 0;
}

// Extract string from args by key
const char *get_string_arg(Node *args, const char *key) {
    for (size_t i = 0; i < args->child_count; i++) {
        if (strcmp(args->keys[i], key) == 0) {
            return args->children[i]->value.string_value;
        }
    }
    return NULL;
}

// Command handlers
void cmd_echo(Node *args, void *data) {
    const char *msg = get_string_arg(args, "message");
    printf("%s\n", msg ? msg : "");
}

void cmd_add(Node *args, void *data) {
    double a = get_number_arg(args, "a");
    double b = get_number_arg(args, "b");
    printf("Result: %g\n", a + b);
}

void cmd_sub(Node *args, void *data) {
    double a = get_number_arg(args, "a");
    double b = get_number_arg(args, "b");
    printf("Result: %g\n", a - b);
}

void cmd_mul(Node *args, void *data) {
    double a = get_number_arg(args, "a");
    double b = get_number_arg(args, "b");
    printf("Result: %g\n", a * b);
}

void cmd_div(Node *args, void *data) {
    double a = get_number_arg(args, "a");
    double b = get_number_arg(args, "b");
    if (b == 0) {
        printf("Error: division by zero\n");
        return;
    }
    printf("Result: %g\n", a / b);
}

void cmd_help(Node *args, void *data) {
    printf("Available commands:\n");
    printf("  echo, add, sub, mul, div, parse, evaluate, help, exit\n");
}

void cmd_parse(Node *args, void *data) {
    const char *json = get_string_arg(args, "json");
    if (json) {
        Node *result = json_parse(json);
        printf("Parsed successfully\n");
        json_free(result);
    }
}

void cmd_exit(Node *args, void *data) {
    printf("Goodbye!\n");
```

## 4. REPL Loop

**Goal:** Main loop that reads, parses, executes.

**Why it matters:** The REPL is what makes it interactive.

**Hints:**
- Read JSON from terminal
- Parse with your JSON parser
- Dispatch command
- Print result or error
- Loop until exit

**Try it:** Create the main loop.

<details>
<parameter name="Show Solution</summary>

```c
#include <stdio.h>
#include <string.h>

void handle_input(const char *input) {
    // Parse JSON
    Node *cmd = json_parse(input);
    if (cmd == NULL) {
        printf("Error: invalid JSON\n");
        return;
    }
    
    // Find "cmd" field
    const char *cmd_name = NULL;
    Node *cmd_args = NULL;
    
    for (size_t i = 0; i < cmd->child_count; i++) {
        if (strcmp(cmd->keys[i], "cmd") == 0) {
            cmd_name = cmd->children[i]->value.string_value;
        }
        if (strcmp(cmd->keys[i], "args") == 0) {
            cmd_args = cmd->children[i];
        }
    }
    
    // Dispatch
    if (cmd_name) {
        for (int i = 0; commands[i].name; i++) {
            if (strcmp(cmd_name, commands[i].name) == 0) {
                commands[i].handler(cmd_args, NULL);
                break;
            }
        }
    }
    
    json_free(cmd);
}

// Modified main loop
int main(void) {
    printf("JSON Toolchain REPL\n");
    printf("Type {'cmd': 'help'} for commands\n> ");
    
    char input[4096];
    while (fgets(input, sizeof(input), stdin) != NULL) {
        // Remove newline
        input[strcspn(input, "\n")] = '\0';
        
        handle_input(input);
        printf("> ");
    }
    
    return 0;
}
```

**Why it matters:** Users make mistakes; tool should handle them.

**Hints:**
- Check json_parse return (NULL on error)
- Handle unknown commands
- Division by zero
- Missing required args

```c
void handle_input(const char *input) {
    // Skip empty input
    if (strlen(input) == 0) return;
    
    // Skip whitespace-only
    int all_space = 1;
    for (int i = 0; input[i]; i++) {
        if (!isspace(input[i])) { all_space = 0; break; }
    }
    if (all_space) return;
    
    // Try to parse
    Node *cmd = json_parse(input);
    if (cmd == NULL) {
        printf("Error: invalid JSON at line %d\n", get_error_line());
        return;
    }
    
    // Check required fields
    if (!has_field(cmd, "cmd")) {
        printf("Error: missing 'cmd' field\n");
        json_free(cmd);
        return;
    }

}
```

## 7. Extension Ideas

Once the basic toolchain works, extend it:

- **Variables** — Store and recall values: `{"cmd": "set", "args": {"name": "x", "value": 5}}`
- **Expressions** — Parse and evaluate: `{"cmd": "eval", "args": {"expr": "x + 1"}}`
- **History** — Arrow keys for command history (readline)
- **Completions** — Tab completion for commands

---

## Final Tips

- **Test incrementally** — Get echo working first, then add commands
- **Print debugging** — Print parsed JSON to verify structure
- **Clean API** — Extract args cleanly to simplify handlers
- **Graceful errors** — Users will make mistakes; handle them well

If your toolchain can handle `{"cmd": "add", "args": {"a": 1, "b": 2}}` and print `Result: 3`, you've built a working language toolchain!
