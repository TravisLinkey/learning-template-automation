# 0.1 — Project 4: Terminal Emulator (PTY)

Build a working terminal emulator that creates a PTY, spawns a shell, and handles I/O between the user and the shell.

## Objective

Create a command-line terminal emulator that:
1. Creates a pseudo-terminal (PTY) pair
2. Spawns a shell (`/bin/bash`) in the PTY slave
3. Reads keyboard input and sends to the shell via PTY
4. Reads shell output from PTY and displays to screen
5. Handles signals for clean shutdown

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    Your Terminal Emulator                    │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐ │
│  │   Keyboard   │───▶│   PTY Master  │───▶│  Shell/bash  │ │
│  │   (stdin)    │    │   (your fd)  │    │  (child)     │ │
│  └──────────────┘    └──────────────┘    └──────────────┘ │
│         ▲                   │                                 │
│         │                   ▼                                 │
│  ┌──────────────┐    ┌──────────────┐                      │
│  │    Screen    │◀───│   PTY Master  │                      │
│  │   (stdout)   │    │   (your fd)   │                      │
│  └──────────────┘    └──────────────┘                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## Focus Areas

- **PTY Management** — Creating and configuring pseudo-terminals
- **Process Control** — Fork/exec for spawning shell
- **I/O Multiplexing** — Using select() for simultaneous I/O
- **Signal Handling** — Graceful shutdown on Ctrl+C
- **Resource Cleanup** — Proper file descriptor and memory management

## Technologies

- **C** — Core implementation language
- **Linux PTY API** — `openpty()`, `ioctl()`, `ptsname()`
- **POSIX API** — `fork()`, `exec()`, `select()`, `signal()`

## Skills You'll Learn

1. Creating PTY pairs with `openpty()`
2. Spawning processes with `fork()` + `exec()`
3. Setting up controlling terminals with `setsid()` and `ioctl()`
4. Using `select()` for event-driven I/O
5. Manipulating terminal attributes with `termios`
6. Handling signals for clean shutdown
7. Proper resource cleanup with `atexit()`

## Deliverable

A working terminal emulator (`terminal`) that:
- Creates a PTY master/slave pair
- Spawns `/bin/bash` in the slave
- Reads from stdin, writes to PTY master
- Reads from PTY master, writes to stdout
- Handles SIGINT (Ctrl+C) gracefully
- Cleans up all resources on exit

## Steps to Complete the Project

### 1. Set up the environment
Create a project folder and compile basic C code.

**Hints:**
- Use a `Makefile` for building
- Compile with: `gcc -Wall -Wextra -o terminal terminal.c`
- Test basic C compilation first

### 2. Create PTY pair
Open a pseudo-terminal pair.

**Hints:**
- Use `openpty(&master, &slave, NULL, NULL, NULL)` from `<pty.h>`
- Save the master fd for reading/writing
- Get slave name with `ptsname(master)` for debugging

### 3. Spawn shell in PTY
Fork and execute bash in the slave.

**Hints:**
- Use `fork()` to create child process
- In child: `close(master)`, set up slave, then `execl()`
- `setsid()` creates new session (becomes session leader)
- `ioctl(slave, TIOCSCTTY, 0)` makes slave the controlling terminal
- Redirect stdin/stdout/stderr to slave with `dup2()`

### 4. Set up terminal for input
Configure terminal for raw mode input.

**Hints:**
- Use `tcgetattr()` to get current settings
- `cfmakeraw()` sets raw mode
- `tcsetattr()` applies new settings
- Save original settings for cleanup!

### 5. Main I/O loop
Read from stdin and PTY, write to PTY and stdout.

**Hints:**
- Use `select()` to wait on both stdin and PTY master
- `FD_SET()` adds fds to watch set
- Read from whichever is ready
- Use non-blocking or check return value of `select()`

### 6. Signal handling
Handle Ctrl+C gracefully.

**Hints:**
- Use `signal(SIGINT, handler)` for cleanup
- Or use a volatile flag checked in the loop
- Restore terminal settings on exit!

### 7. Cleanup
Free all resources properly.

**Hints:**
- Use `atexit(cleanup)` for automatic cleanup
- Close master fd
- Wait for child process with `waitpid()`
- Restore terminal settings with `tcsetattr()`
