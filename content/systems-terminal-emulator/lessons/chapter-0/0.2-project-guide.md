## 2. Create PTY pair

**Goal:** Open a pseudo-terminal pair and print the file descriptors.

**Why it matters:** PTY is the core of a terminal emulator. Without it, you can't talk to the shell.

**Hints:**
- Include `<pty.h>` for `openpty()`
- `int openpty(int *amaster, int *aslave, char *name, struct termios *termp, struct winsize *winp);`
- The function returns 0 on success, -1 on failure
- Use `ptsname(master)` to get the slave device name (for debugging)

**Try it:** Write code that creates a PTY and prints the master/slave fds.

<details>
<summary>Show Solution</summary>

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pty.h>

int main(void) {
    int master, slave;
    char *slave_name;
    
    if (openpty(&master, &slave, NULL, NULL, NULL) < 0) {
        perror("openpty failed");
        return 1;
    }
    
    slave_name = ptsname(master);
    printf("PTY created: master=%d, slave=%d (%s)\n", 
           master, slave, slave_name);
    
    close(master);
    close(slave);
    return 0;
}
```

Output:
```
PTY created: master=3, slave=4 (/dev/pts/1)
```

## 3. Spawn shell in PTY

**Goal:** Fork a child process that runs `/bin/bash` in the PTY slave.

**Why it matters:** The shell is what interprets commands. Without spawning it, there's nothing to talk to.

**Hints:**
- Fork first, then use `openpty()` (or create PTY before fork)
- In child: close master fd
- `setsid()` creates new session → process becomes session leader
- `ioctl(slave, TIOCSCTTY, 0)` makes slave the controlling terminal
- Use `dup2()` to redirect stdin/stdout/stderr to slave
- `execl("/bin/bash", "bash", NULL)` runs the shell
- Parent: close slave fd, keep master for communication

**Try it:** Implement fork/pty/exec sequence.

<details>
<summary>Show Solution</summary>

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pty.h>
#include <sys/wait.h>
#include <fcntl.h>

int main(void) {
    int master, slave;
    pid_t pid;
    
    // Open PTY
    if (openpty(&master, &slave, NULL, NULL, NULL) < 0) {
        perror("openpty");
        return 1;
    }
    
    pid = fork();
    
    if (pid < 0) {
        perror("fork");
        return 1;
    }
    
    if (pid == 0) {
        // Child process: set up slave and exec bash
        close(master);
        
        // Create new session (become session leader)
        setsid();
        
        // Set as controlling terminal
        ioctl(slave, TIOCSCTTY, 0);
        
        // Redirect stdin, stdout, stderr to slave
        dup2(slave, 0);
        dup2(slave, 1);
        dup2(slave, 2);
        close(slave);
        
        // Execute shell
        execl("/bin/bash", "bash", NULL);
        
        // Only reached if exec fails
        perror("execl failed");
        _exit(1);
    }
    
    // Parent process
    close(slave);
    printf("Shell spawned: pid=%d, master_fd=%d\n", pid, master);
    
    // Wait for shell to exit
    waitpid(pid, NULL, 0);
    close(master);
    
    return 0;
}
```

Run it—you should see a shell prompt! Exit with `exit`.

</details>

## 4. Set up terminal for input

**Goal:** Configure the local terminal for raw mode input.

**Why it matters:** Canonical mode (line buffering) delays input until Enter. Raw mode sends each keystroke immediately.

**Hints:**
- Use `tcgetattr()` to get current settings
- `cfmakeraw(&termios)` sets raw mode
- Use `tcsetattr()` to apply
- Save original settings for cleanup!
- Use `TCSANOW` for immediate change or `TCSAFLUSH` to flush input

**Try it:** Add terminal setup before the main loop.

<details>
<summary>Show Solution</summary>

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <termios.h>

static struct termios orig_termios;

void cleanup_terminal(void) {
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &orig_termios);
}

int main(void) {
    struct termios raw;
    
    // Save original terminal settings
    if (tcgetattr(STDIN_FILENO, &orig_termios) < 0) {
        perror("tcgetattr");
        return 1;
    }
    
    // Register cleanup
    atexit(cleanup_terminal);
    
    // Get current settings
    tcgetattr(STDIN_FILENO, &raw);
    
    // Apply raw mode
    cfmakeraw(&raw);
    
    // Set new settings
    if (tcsetattr(STDIN_FILENO, TCSAFLUSH, &raw) < 0) {
        perror("tcsetattr");
        return 1;
    }
    
    printf("Terminal in raw mode. Press Enter to exit...\n");
    getchar();
    
    return 0;
}
```

When you run this, Enter should exit immediately (not wait for a full line).

## 5. Main I/O loop

**Goal:** Read from both stdin and PTY, write to the other.

**Why it matters:** This is the core of the terminal emulator—multiplexing input and output.

**Hints:**
- Use `select()` to wait on both stdin and PTY master
- `FD_ZERO()`, `FD_SET()` to set up file descriptor sets
- Check which fd is ready with `FD_ISSET()`
- Read from ready fd, write to the other
- Handle EOF (when shell exits)

**Try it:** Add the select loop to the PTY code.

<details>
<summary>Show Solution</summary>

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/select.h>
#include <pty.h>
#include <sys/wait.h>
#include <termios.h>

static struct termios orig_termios;
static int master_fd = -1;
static pid_t child_pid = -1;

void cleanup(void) {
    if (master_fd >= 0) close(master_fd);
    if (child_pid > 0) waitpid(child_pid, NULL, 0);
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &orig_termios);
}

int main(void) {
    int master, slave;
    fd_set readfds;
    
    // Save and configure terminal
    tcgetattr(STDIN_FILENO, &orig_termios);
    atexit(cleanup);
    
    struct termios raw = orig_termios;
    cfmakeraw(&raw);
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &raw);
    
    // Create PTY and fork
    openpty(&master, &slave, NULL, NULL, NULL);
    child_pid = fork();
    
    if (child_pid == 0) {
        close(master);
        setsid();
        ioctl(slave, TIOCSCTTY, 0);
        dup2(slave, 0);
        dup2(slave, 1);
        dup2(slave, 2);
        close(slave);
        execl("/bin/bash", "bash", NULL);
        _exit(1);
    }
    
    close(slave);
    master_fd = master;
    
    // Main loop
    while (1) {
        FD_ZERO(&readfds);
        FD_SET(STDIN_FILENO, &readfds);
        FD_SET(master, &readfds);
        
        int maxfd = (master > STDIN_FILENO) ? master : STDIN_FILENO;
        int ret = select(maxfd + 1, &readfds, NULL, NULL, NULL);
        
        if (ret <= 0) continue;
        
        // stdin -> PTY
        if (FD_ISSET(STDIN_FILENO, &readfds)) {
            char buf[4096];
            ssize_t n = read(STDIN_FILENO, buf, sizeof(buf));
            if (n <= 0) break;
            write(master, buf, n);
        }
        
        // PTY -> stdout
        if (FD_ISSET(master, &readfds)) {
            char buf[4096];
            ssize_t n = read(master, buf, sizeof(buf));
            if (n <= 0) break;
            write(STDOUT_FILENO, buf, n);
        }
    }
    
    return 0;
}
```

This should now be a functional terminal emulator!

## 6. Signal handling

**Goal:** Handle signals (especially SIGINT) for graceful shutdown.

**Why it matters:** Without signal handling, Ctrl+C won't work properly and terminal settings won't be restored.

**Hints:**
- Use `signal(SIGINT, handler)` for Ctrl+C
- Or check a volatile flag in the main loop
- Make sure cleanup runs on signal too!

**Try it:** Add signal handlers.

<details>
<summary>Show Solution</summary>

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <signal.h>
#include <sys/select.h>
#include <pty.h>
#include <sys/wait.h>
#include <termios.h>

static struct termios orig_termios;
static volatile sig_atomic_t keep_running = 1;
static int master_fd = -1;
static pid_t child_pid = -1;

void cleanup(void) {
    if (master_fd >= 0) close(master_fd);
    if (child_pid > 0) kill(child_pid, SIGTERM);
    if (child_pid > 0) waitpid(child_pid, NULL, 0);
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &orig_termios);
}

void signal_handler(int sig) {
    keep_running = 0;
}

int main(void) {
    int master, slave;
    fd_set readfds;
    
    // Setup signals
    signal(SIGINT, signal_handler);
    signal(SIGTERM, signal_handler);
    signal(SIGCHLD, signal_handler);
    
    // Save and configure terminal
    tcgetattr(STDIN_FILENO, &orig_termios);
    atexit(cleanup);
    
    struct termios raw = orig_termios;
    cfmakeraw(&raw);
    tcsetattr(STDIN_FILENO, TCSAFLUSH, &raw);
    
    // Create PTY and fork
    openpty(&master, &slave, NULL, NULL, NULL);
    child_pid = fork();
    
    if (child_pid == 0) {
        close(master);
        setsid();
        ioctl(slave, TIOCSCTTY, 0);
        dup2(slave, 0);
        dup2(slave, 1);
        dup2(slave, 2);
        close(slave);
        execl("/bin/bash", "bash", NULL);
        _exit(1);
    }
    
    close(slave);
    master_fd = master;
    
    // Main loop
    while (keep_running) {
        FD_ZERO(&readfds);
        FD_SET(STDIN_FILENO, &readfds);
        FD_SET(master, &readfds);
        
        struct timeval tv = {1, 0};  // 1 second timeout
        int maxfd = (master > STDIN_FILENO) ? master : STDIN_FILENO;
        int ret = select(maxfd + 1, &readfds, NULL, NULL, &tv);
        
        if (ret < 0) {
            if (keep_running) perror("select");
            break;
        }
        if (ret == 0) continue;  // Timeout
        
        // stdin -> PTY
        if (FD_ISSET(STDIN_FILENO, &readfds)) {
            char buf[4096];
            ssize_t n = read(STDIN_FILENO, buf, sizeof(buf));
            if (n <= 0) break;
            write(master, buf, n);
        }
        
        // PTY -> stdout
        if (FD_ISSET(master, &readfds)) {
            char buf[4096];
            ssize_t n = read(master, buf, sizeof(buf));
            if (n <= 0) break;
            write(STDOUT_FILENO, buf, n);
        }
    }
    
    return 0;
}
```

Now Ctrl+C works and terminal settings are properly restored.

## 7. Final clean-up and polish

**Why it matters:** Proper cleanup prevents resource leaks and leaves the terminal in a usable state.

## 8. Final tips for beginners

- **Start simple.** Get PTY creation working first, then add fork/exec, then I/O loop.
- **Print debugging.** Add `printf()` statements to track execution flow.
- **Test incrementally.** Each step should compile and run before adding the next.
- **Check return values.** `openpty()`, `fork()`, `select()` can all fail—handle errors.
- **Use `strace`** (Linux) or `dtruss` (macOS) to trace system calls if stuck.

If you can run this terminal emulator and use bash inside it, you've successfully built a real terminal emulator from scratch!
